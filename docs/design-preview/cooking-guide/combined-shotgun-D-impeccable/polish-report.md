## Drift diagnosis

1. **Missing token system**: D used a self-contained hex palette with no shared CookPilot tokens. The combined result aligns to the OKLCH tokens established by the independent Impeccable product-only path.
2. **Conceptual hierarchy misalignment**: voice state was visually secondary even though the product requires users to trust whether the app is listening. It is promoted to a full-width, high-contrast live status.
3. **Incomplete content model**: the middle surface assumed timer-only content, but planned cooking media changes that surface's role. It now has an image slot with a visible completion cue.
4. **One-off control hierarchy**: D split one primary action from three utilities. The user preferred A's four-command deck, so the combined result restores four equally reachable command locations while keeping `다음` visually primary.
5. **Timer exploration**: the oversized free-standing digits become a ring-plus-digits instrument, making progress visible without relying on color alone.
